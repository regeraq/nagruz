## 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема 1: Таблица не существует

**Симптомы:**
- Ошибка в логах: "Table 'commercial_proposal_files' does not exist"
- Файлы не сохраняются при отправке формы

**Решение:**
```bash
# Применить миграцию
npm run db:push

# Или создать таблицу вручную
psql -U loaddevice_user -d loaddevice_db -f create_commercial_proposal_files.sql
```

### Проблема 2: Ошибки при загрузке файлов

**Симптомы:**
- Ошибка 500 при отправке формы
- Файлы не загружаются

**Решение:**
1. Проверить логи: `pm2 logs loaddevice --err`
2. Проверить размер файлов (максимум 10 МБ на файл, 50 МБ общий)
3. Проверить формат файлов (только PDF, DOC, DOCX, XLS, XLSX)

### Проблема 3: Мониторинг БД не работает

**Симптомы:**
- Ошибка загрузки данных в админ-панели
- Не отображается информация о размере БД

**Решение:**
1. Проверить права доступа пользователя БД
2. Проверить логи: `pm2 logs loaddevice --err`
3. Проверить, что endpoint `/api/admin/database/size` доступен

### Проблема 4: Приложение не запускается

**Симптомы:**
- PM2 показывает статус "errored" или "stopped"
- Ошибки в логах

**Решение:**
```bash
# Проверить статус
pm2 status

# Просмотреть логи
pm2 logs loaddevice --err --lines 100

# Перезапустить
pm2 restart loaddevice

# Если не помогает, пересобрать
npm run build
pm2 restart loaddevice
```
