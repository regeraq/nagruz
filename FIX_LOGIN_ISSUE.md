# 🔧 Исправление проблемы с входом в систему

## Проблема
После обновления кода пользователи не могут войти, получая ошибку 403 "Ваш аккаунт заблокирован".

## Возможные причины и решения

### 1. Пользователи заблокированы в базе данных

**Решение:** Выполните SQL скрипт для разблокировки:

```bash
# Подключитесь к вашей PostgreSQL базе данных
psql -U your_user -d your_database

# Или используйте pgAdmin / другой клиент БД
```

Затем выполните:

```sql
-- Разблокировать всех пользователей
UPDATE users SET is_blocked = false WHERE is_blocked = true;

-- Проверить результат
SELECT id, email, role, is_blocked FROM users;
```

Или используйте готовый скрипт:
```bash
psql -U your_user -d your_database -f fix-blocked-users.sql
```

---

### 2. Проблема с CSRF токеном (если сайт на HTTP)

**Проблема:** Если сайт работает по HTTP (не HTTPS), а в production включены secure cookies, CSRF токен не может быть установлен.

**Решение:** Убедитесь, что в `.env` файле:

```env
# Если сайт на HTTP (не HTTPS), установите:
FORCE_SECURE_COOKIES=false

# Или если за reverse proxy с HTTPS:
FORCE_SECURE_COOKIES=true
```

---

### 3. JWT_SECRET изменился

**Проблема:** Если `JWT_SECRET` изменился, все старые токены не работают.

**Решение:** 
- Убедитесь, что `JWT_SECRET` в `.env` тот же, что был раньше
- Или все пользователи должны выйти и войти заново

---

## Диагностика

### Шаг 1: Проверьте статус пользователя

Откройте в браузере (замените `your-email@example.com` на реальный email):

```
http://your-server/api/debug/user-status/your-email@example.com
```

Это покажет:
- Существует ли пользователь
- Заблокирован ли он (`isBlocked`)
- Его роль

### Шаг 2: Проверьте логи сервера

При попытке входа в логах должно быть:

```
[Login] User found: email@example.com, isBlocked: false, role: admin
```

Если видите `isBlocked: true` - пользователь заблокирован в БД.

### Шаг 3: Проверьте CSRF токен

В консоли браузера (F12) проверьте:
- Есть ли cookie `csrf-token`
- Отправляется ли заголовок `x-csrf-token` в запросах

---

## Быстрое исправление

### Вариант 1: Разблокировать всех пользователей

```sql
UPDATE users SET is_blocked = false;
```

### Вариант 2: Разблокировать только администраторов

```sql
UPDATE users SET is_blocked = false WHERE role IN ('admin', 'superadmin');
```

### Вариант 3: Разблокировать конкретного пользователя

```sql
UPDATE users SET is_blocked = false WHERE email = 'your-email@example.com';
```

---

## После исправления

1. **Перезапустите сервер:**
   ```bash
   # Если используете PM2
   pm2 restart your-app
   
   # Или если напрямую
   npm run start
   ```

2. **Очистите кэш браузера** и попробуйте войти снова

3. **Проверьте логи** - должны быть сообщения о успешном входе

---

## Если проблема не решена

1. Проверьте логи сервера на наличие ошибок
2. Убедитесь, что база данных доступна
3. Проверьте, что все переменные окружения установлены правильно
4. Убедитесь, что сервер перезапущен после изменений

---

## Удаление диагностического эндпоинта

После исправления проблемы удалите временный эндпоинт `/api/debug/user-status/:email` из `server/routes.ts` для безопасности.
